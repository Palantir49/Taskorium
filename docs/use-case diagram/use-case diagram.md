Три роли пользователей

1. Исполнитель (Assignee)
Кто это: Сотрудник, который выполняет задачи
Что может делать:

Войти/выйти из системы, сбросить пароль

Просматривать назначенные ему задачи

Изменять статус выполнения задач (например: "В работе", "Готово")

Добавлять комментарии к задачам

Прикреплять файлы к задачам

Просматривать проекты, в которых участвует

Смотреть и редактировать свой профиль

Получать уведомления о новых задачах и дедлайнах

Просматривать историю изменений задач

Не может: создавать задачи, назначать их другим, управлять проектами

2. Менеджер (Manager)
Кто это: Руководитель, который управляет задачами и проектами
Что может делать (ВСЕ, что может Исполнитель, плюс):

Создавать новые задачи

Редактировать существующие задачи

Удалять задачи

Устанавливать приоритеты задач

Назначать задачи Исполнителям

Создавать проекты

Управлять участниками проектов

Устанавливать сроки выполнения проектов

Закрывать проекты

Просматривать статистику по работе

Создавать отчеты по задачам

Экспортировать данные

Анализировать продуктивность команды

Не может: управлять системными настройками, назначать роли пользователям

3. Администратор (Administrator)
Кто это: Технический специалист, который управляет всей системой
Что может делать (ВСЕ, что может Менеджер, плюс):

Регистрировать новых пользователей

Управлять ролями пользователей (назначать кого Менеджером, кого Исполнителем)

Настраивать систему (параметры, интеграции)

Просматривать логи системы (журналы действий)

Делать резервные копии данных

Управлять внешними интеграциями (например, с email-сервисом)
